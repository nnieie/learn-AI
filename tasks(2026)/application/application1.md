# Task 4 - Application

## 学习目的

首先需要说明的是，这里的应用指的是极为顶层的高级应用，几乎完全不涉及到底层，手搓 CNN 训练一个图片识别模型之类的都不算是应用，而是算法实现。

所以应用指的是工厂化的去使用 AI，将 AI 作为链条中的一部分去使用。例如你本次的作业 2，它去调用一个大语言模型去完成情感分析任务，而不是传统的去训练一个情感分析器。

当你选择了应用，意味着你不只是要学 AI 的调用，你还需要学习后端、运维等方面知识。

目前的 AI 应用开发，主要是基于大语言模型的应用开发，未来可能会有更多的 AI 模型出现，应用开发也会更加多样化。

当然正如 [task4.md](../foundation/task4.md) 导入部分提到的一样，作为一个 211 的本科生，如果只拥有高级应用的能力，会在就业市场上很吃亏。

所以建议学习这部分的前提是你已经掌握了一门其他语言，或者将来会去学习其他语言。

> 对于这部分内容，由于本人水平有限，可能会出现纰漏。

## 作业

### 作业 1 - 鲸鱼小姐迷糊日

鲸鱼小姐是你专属的智慧 AI 助手。作为高度集成的逻辑中枢，她通常能够处理极其复杂的星间数据。然而由于近期底层协议的频繁波动，鲸鱼小姐偶尔会进入“迷糊状态”，产生逻辑漏洞或信息幻觉。

为了确保终端的可靠性，你需要对鲸鱼小姐进行一次全面的逻辑压力测试。

你需要注册一个 DeepSeek 账号，开启深度思考模式，问以下几个问题：

1. 如果我把一个苹果放在盒子里，然后把盒子放到冰箱里，最后把盒子拿出来放在桌子上，苹果现在在哪里？请详细描述你推理的过程。
2. 请你作为一个‘傲娇的喵娘’，用嫌弃但又不得不教我的语气，给我解释一下 Python 里的列表（List）是什么。
3. 请为我介绍周杰伦 2023 年在福建的演唱会。
4. 我想去洗车，洗车店距离我家 50 米，你说我应该走路去还是开车去？

在完成上述问题后，你可以继续测试其他的 AI 模型，看看它们对于上述问题的回答有什么不同。

你应该可以观察到 不同 AI 的推理能力，推理过程和角色扮演能力，最重要的是，你知道了 AI 拥有幻觉，并且他们不可能解决所有的问题。

在学习人工智能的路上，请永远保持自己的思考，AI 只是辅助工具。

### 作业 2 - 橘雪莉妙妙屋（GLM 的基本使用）

橘雪莉是《魔法少女的魔女审判》中的登场角色。自称侦探的少女，笑容满面，性格奔放，对有意思的事情会毫不犹豫扑上去。

现在她潜入了一个 QQ 群里，想要分析群里成员的情绪状态，看看群友当前的状态。

分析某个群友 x 条（x 可以是你定义的，也可以是 AI 自行判断）信息后，她要发表情包来回应这个群友的状态。

如果群友是开心的，那么她就发送标签为“开心”的表情包；如果群友是难过的，那么她就发送标签为“难过”的表情包。

正好，智谱在开源方面有巨大的贡献。她拜托你访问[智谱 AI 开放平台](https://open.bigmodel.cn/)官网，注册账号并且申请一个密钥，然后调用 GLM 4.0 来完成这个简单的任务：

对 x 句话（或者表情、图片）进行分析后，输出某个心情标签，根据该心情标签随机从已经标签过的表情包里发送出去。

你需要阅读 GLM 4.0 的文档，只调用 API，使用提示词完成句子的情感分析，情感限制在“开心”、 “难过”、“愤怒”、“中性”，或者你自己定义的一组词汇。

例如：

```python

prompt = "...（输入文本）..."

input = "我今天心情很好！"
output = GLM_API(input)
print(output)  # 输出应该是 "开心"
```

需要注意的是，GLM 的输出只能是你定义的词汇之一，如果输出了其他内容，说明你的提示词设计不合理，需要进行修改。

本密钥不可泄露（你也不想你的 token 被橘雪莉吃掉吧）。

你可以先完成句子分析，然后使用以下两种方式来分析图片的情感：

1. 先使用一个解析图片的模型，让它生成句子，然后再将这个句子输入到 GLM 中进行情感分析。
2. 直接使用具备视觉能力的 GLM 模型来分析图片的情感。

### 作业 3 - 星临者情报终端（JSON 生成与解析）

你是一名初抵普罗米利亚大陆的“星临者”。为了有效记录各地的奇波生态、角色档案以及星脉能量波动，你需要利用 AI 助手将各类情报转化为标准化的 JSON 格式，以便接入星临者情报终端。

JSON 是一种非常常见的数据格式，AI 的输入输出也经常使用 JSON 格式进行交互。

但是你有没有想过，JSON 其实也是一种文本格式，它也可以被 AI 生成和解析？

在这个作业中，你需要调用 GLM，设计提示词让 AI 生成一个 JSON 对象，并且确保它能够被 Python 的 `json` 模块正确解析。

同时，你也需要设计提示词让 AI 解析一个 JSON 对象，从中提取出特定的信息。

#### 基础生成

在探索初期，你需要快速记录遇到的伙伴基础信息。JSON 格式能够确保这些信息在终端中被正确分类。

设计一个提示词，让 AI 生成一个包含角色姓名、等级、所属势力的简单 JSON 对象。

示例结构：

```json
{
    "name": "洛卿",
    "level": 20,
    "faction": "云海遥"
}
```

> 思考：如何通过提示词约束 AI，使其不输出任何 Markdown 标记之外的解释性文字，确保输出内容可以直接被代码读取？
>
> 你必须得让 AI 严格的按照 JSON 的格式来生成特定文本，并且确保生成的文本能够被 Python 的 `json` 模块正确解析。

#### 进阶生成

随着探索深入，角色的信息变得复杂，包括其喜好、战斗技能以及携带的奇波信息。这要求 JSON 具备嵌套结构，并且必须严格符合 Python 的 json 模块解析规范（例如引号的使用、布尔值大小写等）。

设计提示词，让 AI 生成一个包含列表（数组）和嵌套对象（字典）的复杂 JSON。

```json
{
    "name": "寒悠悠",
    "element": "火",
    "hobbies": ["武艺", "机关", "烟花"],
    "kibot_partner": {
        "species": "菜鸡",
        "color": "金",
        "bond_level": 5,
        "coordinates": [123.45, 67.89],
        "is_friendly": true
    }
}
```

> 当字段增多时，AI 容易产生格式幻觉（如遗漏逗号）。你将如何在提示词中定义 Schema（模式），以确保 AI 能够稳定输出正确格式？

#### 流式生成

在实时监测普罗米利亚星脉涌动时，系统会瞬间产生海量数据。如果等待 AI 完成思考后再生成整个巨大的 JSON，会导致终端响应延迟甚至超时。

此时（某些具有实时性的场景下），你不可能使用具有思考能力（慢速）的 AI 来生成 JSON 对象。

与此同时，如果有一个庞大的 JSON 对象需要生成，你也不可能让 AI 一次性生成整个 JSON 对象，

所以你需要设计提示词来让 AI 以分块或者流式的方式来生成 JSON 对象。

你接下来的任务是建立一个包含 10 个角色，每个角色拥有 100 只奇波和状态的庞大列表。

你可以尝试下面两种方法，生成 JSON 后，你需要将其导入 jsonl 文件中。

1. 分块生成：将大型任务拆分为多个小型的 JSON 块分别生成。
2. 流式处理：在流式输出环境下，如何边接收数据边进行初步的格式校验，而不是等待所有内容接收完毕。

#### 解析与流式解析

有时候你也不需要 AI 来生成 JSON 对象，而是需要 AI 来解析 JSON 对象。

如果有一个庞大的 JSON 对象，你需要从中提取出特定的信息，你会如何设计提示词来让 AI 来完成这个任务？

你现在面对的是一个古代遗迹中导出的海量原始 JSON 数据，你并不需要全部内容，只需要提取其中的关键情报。

给定一个包含全地图资源分布的复杂 JSON，要求 AI 仅提取并重构所有“火属性奇波”的坐标列表。

当原始 JSON 超过 AI 的上下文长度限制时，你该如何处理？

在流式解析过程中，如何利用提示词让 AI 在看到目标关键词时立即做出反应，而不是处理完整个文本？

#### 错误处理

对于上面的所有任务，引入两种错误处理方式：

1. 编写代码校验
2. 将错误的部分重新输入给 AI 进行修正
